# Style Guide

This document outlines the coding style and conventions for the IriSync-Rewrite project.

## General Principles

*   **Clarity:** Write clear and concise code that is easy to understand and maintain.
*   **Consistency:** Adhere to the established coding style and conventions throughout the project.
*   **Simplicity:** Favor simple solutions over complex ones.

## Naming Conventions

*   **Variables:** Use descriptive, camelCase variable names.
*   **Functions:** Use descriptive, camelCase function names.
*   **Classes:** Use descriptive, PascalCase class names.
*   **Interfaces:** Use descriptive, PascalCase interface names, prefixed with an "I".

## Formatting

*   **Indentation:** Use 2 spaces for indentation.
*   **Line Breaks:** Use line breaks to improve readability.
*   **Comments:** Use comments to explain complex or non-obvious code.

## Best Practices

*   **Error Handling:** Implement robust error handling to prevent unexpected crashes.
*   **Testing:** Write unit tests for all new features and bug fixes.
*   **Documentation:** Document all public APIs and complex algorithms.
